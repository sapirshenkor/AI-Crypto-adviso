# Phase Completion Checklist

Before finishing any phase:

1. Confirm the requested phase only was implemented.
2. Do not introduce unrelated features.
3. Check that existing behavior still works.
4. Update `.env.example` if new environment variables were added.
5. Add or update minimal documentation if needed.
6. Summarize:
   - files changed
   - what was implemented
   - how to test manually
   - recommended commit message
